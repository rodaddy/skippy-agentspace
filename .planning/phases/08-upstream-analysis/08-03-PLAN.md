**Rewrite `skills/skippy/commands/update.md`** as an AI-driven markdown command (not a shell script wrapper). The command should describe INTENT and let Claude adapt -- per RESEARCH.md Pitfall 4, avoid making it ">50% code blocks."

Structure:
```
---
name: skippy:update
description: Check all tracked upstreams for changes and suggest cherry-picks
---
```

The command must include these process steps (as described in RESEARCH.md):

1. **Discover Upstreams** -- Read all `upstreams/*/upstream.json` files from the skippy-agentspace repo root. Build a list of {name, repo, branch, last_checked_sha, cherry_picks}. NEVER hardcode repo URLs -- iterate the directory.

2. **Check Each Upstream** -- For each upstream: determine cache directory (`~/.cache/skippy-upstream/<name>`), clone if not cached or fetch if cached, get current HEAD SHA, compare against last_checked_sha.

3. **Report Changes** -- For upstreams with changes: show commit count, date range, changed files grouped by area, highlight changes in cherry_picks areas, suggest potential new cherry-picks. For upstreams with no changes: brief "no changes since <date>" line.

4. **Update Tracking** -- For each checked upstream, update its upstream.json in-place: set last_checked_sha to current HEAD, set last_check to today's date (ISO 8601).

5. **Cross-Package Analysis Flag** -- Check if `docs/cross-package-analysis.md` exists. If any upstream has significant changes (>10 commits or changes in cherry-picked areas), suggest re-reviewing the analysis document. Note its 'last reviewed' date.

Key constraints:
- The command is AI-driven: describe what to do conversationally, not as bash commands Claude runs mechanically
- No auto-merge -- present findings and let the user decide
- Generic iteration: works for 2 upstreams or 20 with no command changes
- Include an `<execution_context>` referencing `@../SKILL.md`
