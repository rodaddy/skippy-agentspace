<task type="auto">
  <name>Task 1: Create prereqs.sh -- interactive prerequisite checker</name>
  <files>tools/prereqs.sh</files>
  <action>
Create `tools/prereqs.sh` -- an interactive prerequisite validator that checks for required tools and offers to install missing ones.

**Shebang:** `#!/usr/bin/env bash` with `set -euo pipefail`

**OS detection function (`detect_os`):**
- Use `uname -s` to distinguish Darwin vs Linux
- On Linux, parse `/etc/os-release` `ID=` field to detect ubuntu/debian/pop/mint, fedora/rhel/centos, arch/manjaro/endeavouros
- Detect WSL2 via `grep -qi microsoft /proc/version 2>/dev/null`
- Set `OS`, `PKG_MGR`, `PKG_INSTALL` variables based on detection
- For unknown OS/distro, set `PKG_MGR="unknown"` and skip auto-install

**Tool checks (4 tools):**

1. **git**: `command -v git`, version via `git --version | cut -d' ' -f3`
2. **bash 4+**: Find PATH bash via `command -v bash`, get version via `"$bash_path" -c 'echo $BASH_VERSION'`, parse major version. FAIL if < 4. Do NOT check the running shell -- check what `bash` resolves to in PATH.
3. **bun**: `command -v bun`, version via `bun --version`
4. **jq**: `command -v jq`, version via `jq --version`

**For each tool, output:**
- If present: `  OK: <tool> <version>` (and for bash, include the path)
- If missing/outdated: `  MISSING: <tool>` or `  OUTDATED: bash <version> (need 4+)`

**Interactive install (for missing tools only):**
- Prompt: `printf "Install %s? [y/N] " "$tool"` then `read -r response`
- Read from `/dev/tty` to support piped execution edge cases
- Accept y/Y/yes/YES
- Platform-specific install commands:
  - bun: macOS `brew install oven-sh/bun/bun`, Linux `curl -fsSL https://bun.sh/install | bash`
  - jq: `$PKG_INSTALL jq`
  - git: `$PKG_INSTALL git`
  - bash: macOS `brew install bash`, Linux `$PKG_INSTALL bash`
- For unknown PKG_MGR, print the install command but don't auto-install: "Install manually: <command>"

**Summary at end:**
- Count OK / MISSING / OUTDATED
- If all OK: "All prerequisites met."
- If any missing: "N tool(s) still missing. Install them and re-run."
- Exit 0 if all met, exit 1 if any missing after prompts

**Anti-patterns to avoid:**
- Do NOT hardcode `/usr/local/bin` or `/opt/homebrew/bin` paths -- use `command -v`
- Do NOT parse `bash --version` output text -- use `$BASH_VERSION` variable via subshell
- Do NOT use `readlink -f` (not available on macOS)
- Do NOT use associative arrays or bash 4+ features in prereqs.sh itself (it may run under bash 3.2 on first use before bash is upgraded)
  </action>

  </verify>
  <done>prereqs.sh detects OS, reports version status for all 4 tools (bun, jq, bash 4+, git), offers interactive install for missing tools, exits 0 when all present</done>

<task type="auto">
  <name>Task 2: Create verify.sh -- reusable health check (brew doctor style)</name>
  <files>tools/verify.sh</files>
  <action>
Create `tools/verify.sh` -- a reusable health check script that validates the entire PAI installation, modeled on `brew doctor`.

**Shebang:** `#!/usr/bin/env bash` with `set -euo pipefail`

**Output formatting (inline, not shared lib -- per research recommendation):**
```
PASS_COUNT=0; WARN_COUNT=0; FAIL_COUNT=0
pass() { echo "  PASS: $1"; PASS_COUNT=$((PASS_COUNT + 1)); }
warn() { echo "  WARN: $1"; WARN_COUNT=$((WARN_COUNT + 1)); }
fail() { echo "  FAIL: $1"; FAIL_COUNT=$((FAIL_COUNT + 1)); }
suggest() { echo "    Fix: $1"; }
```

**Repo root detection:** `REPO_ROOT="$(cd "$(dirname "$0")/.." && pwd)"` (same pattern as validate-hooks.sh)

**Category 1: Prerequisites**
- Check bun, jq, git via `command -v` -- PASS with version or FAIL with install suggestion
- Check bash version (PATH bash, not running shell) -- PASS if 4+, WARN if 3.x with brew install suggestion

**Category 2: Skills**
- Iterate `$REPO_ROOT/skills/*/`
- For each skill, check if symlink exists at `~/.claude/skills/$name` OR `~/.claude/commands/$name`
- If symlink exists, verify target resolves (`-e` after following) -- PASS if resolves, FAIL if dangling with "Re-run tools/install.sh" suggestion
- If no symlink: WARN (not installed) with `tools/install.sh $name` suggestion
- Special case: if `core` skill is not installed, FAIL (not WARN) since core is essential

**Category 3: Hooks**
- Delegate structural checks to `tools/validate-hooks.sh` -- run it, PASS if exit 0, FAIL if exit 1 with "run tools/validate-hooks.sh for details" suggestion
- Check `~/.claude/settings.json` for PAI hook registrations:
  - Use `bun -e` to count hooks with commands containing `skills/core/hooks/` (same pattern as validate-hooks.sh Check 6)
  - 15 hooks = PASS, 0 hooks = WARN (not installed) with install-hooks.sh suggestion, 1-14 = WARN (partial) with re-run suggestion
  - If settings.json missing: WARN with install-hooks.sh suggestion

**Category 4: Commands**
- Check that skippy commands are accessible by verifying symlink for skippy skill
- Check each expected command file exists: reconcile.md, update.md, cleanup.md, migrate.md
- PASS per command found, WARN if missing

**Additional check: INDEX.md consistency**
- Call `bash "$REPO_ROOT/tools/index-sync.sh" --check 2>/dev/null` -- but only if index-sync.sh supports --check. If exit 0 PASS, if exit 1 WARN with "run tools/index-sync.sh" suggestion. If --check not supported, skip gracefully.

**Summary:**
```
=== Summary ===
  X passed, Y warnings, Z failures
```
- Exit 0 if FAIL_COUNT == 0
- Exit 1 if FAIL_COUNT > 0

**Anti-patterns to avoid:**
- Do NOT reimplement hook validation logic -- delegate to validate-hooks.sh
- Do NOT reimplement skill listing -- iterate directories directly
- Do NOT use `readlink -f` -- check symlink with `-L` and target with `-e`
- Do NOT use `local` inside conditionals at top level (bash portability)
  </action>
  <verify>
    <automated>bash tools/verify.sh 2>&1 | head -5 | grep -q "=== Prerequisites ===" && bash tools/verify.sh 2>&1 | tail -3 | grep -q "passed" && echo "verify.sh runs with categories and summary" || echo "FAIL"</automated>
  </verify>
  <done>verify.sh checks 4 categories (Prerequisites, Skills, Hooks, Commands), outputs PASS/WARN/FAIL with fix suggestions, delegates to existing tools where possible, exits 0 when no failures</done>
</task>
