- Run `/skippy:upgrade` -- it handles pull, reinstall, verification, and customization detection
- Explain briefly what it does (pre-snapshot, pull, reinstall, verify, conflict detection)

Section 3: **What gets preserved**
- Custom skills you created (not in upstream) are untouched
- settings.json hooks are merged non-destructively (install-hooks.sh is idempotent)
- Your `~/.claude/settings.json` permissions and non-PAI hooks are never modified

**File 3: skills/skippy/commands/upgrade.md**

Follow the Phase 8 AI command pattern exactly (match structure of `update.md`).

```yaml
---
name: skippy:upgrade
description: Upgrade skippy-agentspace to latest version preserving customizations
---
```

Process steps:
1. **Pre-upgrade snapshot**: Record installed skills (run `tools/install.sh` no-arg output), hook count in settings.json, any modified tracked files (`git status`)
2. **Pull latest**: `cd` to repo root, `git pull origin main` (or current branch), report new commits with `git log --oneline` for the range
3. **Re-install**: Run `bash tools/install.sh --all`, run `bash skills/core/hooks/install-hooks.sh`
4. **Verify**: Run `bash tools/verify.sh`, compare post-upgrade state to pre-upgrade snapshot, report added/removed/updated skills
5. **Handle customizations**: If `git status` shows modified tracked files after pull, report conflicts and suggest resolution options (keep upstream, keep local, manual merge)
