**lib/merge.ts** -- TypeScript module for JSON operations, called by shell scripts via `bun run`.

Accepts CLI args: `bun run merge.ts <action> <settings-path> <manifest-path> <hooks-dir>`

Actions:
- `merge`: Read settings.json and manifest.json. For each manifest hook entry:
  1. Resolve script path to absolute: `<hooks-dir>/<script>` (follow symlinks with `Bun.resolveSync` or `fs.realpathSync`)
  2. Build command string: `bun run <absolute-script-path>`
  3. Check if settings.json already has a hook with this exact command string anywhere under `hooks[event]` -- if yes, skip (idempotency per HOOK-04)
  4. Group manifest hooks by event+matcher combination for efficiency (e.g., all PreToolUse Write|Edit|Bash hooks become one matcher group)
  5. Append new matcher groups to `hooks[event]` array. Create the event array if it doesn't exist.
  6. Write the full settings.json back with `JSON.stringify(obj, null, 2)` -- preserve all non-hook settings
  7. Output summary: "Added N hooks, skipped M (already installed)"

- `remove`: Read settings.json and manifest.json. For each matcher group in settings.json:
  1. Filter the group's `hooks` array, removing entries where command contains the manifest's `identifier` ("skills/core/hooks/") AND the command matches a script in the manifest (double-check per HOOK-03)
  2. If a matcher group's hooks array becomes empty after filtering, remove the entire matcher group
  3. If an event type's array becomes empty, remove the event key
  4. Write settings.json back
  5. Output summary: "Removed N hooks"

CRITICAL: Only modify the `hooks` key in settings.json. Leave ALL other keys untouched. Read the full JSON, modify only hooks, write the full JSON back.

**install-hooks.sh** -- Shell script orchestrating installation flow.

```
#!/usr/bin/env bash
set -euo pipefail
```

Steps:
1. Parse args: `--dry-run` (show what would change without writing), `--settings=PATH` (override settings.json location, default `$HOME/.claude/settings.json`)
2. Validate prerequisites: `command -v bun` (required -- no fallback for install script itself, though merge.ts handles the JSON). If bun not found, print error with install instructions and exit 1.
3. Resolve paths: SCRIPT_DIR (directory of install-hooks.sh), HOOKS_DIR (same as SCRIPT_DIR), MANIFEST (SCRIPT_DIR/manifest.json), SETTINGS (default or override)
4. Validate manifest exists
5. Create backup (HOOK-05): `cp "$SETTINGS" "$SETTINGS.backup-$(date +%Y-%m-%d-%H%M%S)"`. If settings.json doesn't exist yet, create it with `{"hooks":{}}` and skip backup.
6. Run merge: `bun run "$HOOKS_DIR/lib/merge.ts" merge "$SETTINGS" "$MANIFEST" "$HOOKS_DIR"`
7. Report results (backup location, hooks added/skipped)

For `--dry-run`: Run merge.ts with a temp copy of settings.json, show diff, don't write.

**uninstall-hooks.sh** -- Shell script orchestrating removal flow.

Same structure as installer but:
1. Validate settings.json exists (can't uninstall from nothing)
2. Create backup before removal (same timestamped pattern)
3. Run: `bun run "$HOOKS_DIR/lib/merge.ts" remove "$SETTINGS" "$MANIFEST" "$HOOKS_DIR"`
4. Report results (hooks removed, backup location)

Both scripts must be executable (`chmod +x` in the action).

Per user decision: "Shell script handles flow: backup, validation, reporting. Delegates JSON merge to detected backend." The primary backend is bun (merge.ts). The multi-backend fallback (python3, jq) is a nice-to-have -- implement bun-only for now. The shell scripts print a clear error if bun is not available.
