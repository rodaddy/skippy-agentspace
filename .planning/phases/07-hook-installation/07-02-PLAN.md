**LAW 6 -- Interview-First** (PreToolUse, matcher: Write|Edit|Bash)
- Detects when Claude is about to implement a significant feature without first asking clarifying questions
- Check if recent conversation has enough user-provided requirements (look for question marks in assistant messages, user responses to questions)
- Use `askDecision` (not block) -- this is a soft nudge, not a hard gate
- Skip for small edits (files under 20 lines of change), config files, and test files

**LAW 10 -- qmd First** (PreToolUse, matcher: Read|Glob|Grep)
- Warns when Read/Glob/Grep is used before trying qmd MCP search tools
- Check if qmd tools (search, vsearch, query) were recently used in the conversation
- Use `askDecision` -- remind to try qmd first, but don't block if user already chose to use file tools
- Skip when no qmd MCP is available (check if tool was ever used in session)

**LAW 12 -- Private Repos Default** (PreToolUse, matcher: Bash)
- Blocks `gh repo create` commands that don't include `--private`
- Detect: command contains `gh repo create` without `--private` flag
- Use `blockDecision` -- this is a hard gate per LAW
- Allow if command explicitly has `--public` (user override is intentional)

**LAW 13 -- No Silent Autopilot** (UserPromptSubmit, no matcher)
- This is the ONLY UserPromptSubmit hook. Different event type -- no tool_name/tool_input.
- Detects keywords in user prompts that suggest enabling autopilot without safeguards
- Keywords: "do everything", "don't ask", "just do it", "autopilot", "yolo mode", "full auto"
- Use `blockTopLevel` from feedback.ts (NOT blockDecision -- different output format for non-PreToolUse events)
- Note: the input for UserPromptSubmit has a `prompt` field with the user's message text

**LAW 14 -- Network Share Protocol** (PreToolUse, matcher: Bash)
- Blocks raw mount commands for SMB/NFS without following the network share protocol
- Detect: `mount -t smbfs`, `mount -t nfs`, `mount_smbfs`, `mount_nfs` commands
- Use `blockDecision` with instructions to use the network share protocol script instead
- Allow `umount` commands (unmounting is always safe)

Same requirements as Task 1: shebang, subagent skip, fail-open error handling, snake_case fields, imports from ./lib/.
