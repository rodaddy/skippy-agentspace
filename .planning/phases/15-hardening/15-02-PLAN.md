  <name>Task 1: Create bump-version.sh with dry-run and verification</name>
  <files>tools/bump-version.sh</files>
  <action>
Create `tools/bump-version.sh` with `#!/usr/bin/env bash` and `set -euo pipefail`.

**Common.sh sourcing:** Use the established pattern from verify.sh -- `BASH_SOURCE` path derivation with graceful fallback. The fallback needs stubs for: `skippy_repo_root`, `skippy_pass`, `skippy_fail`, `skippy_section`, `skippy_summary`. Use `$0` (not `$BASH_SOURCE`) in the fallback stub for `skippy_repo_root` per project gotcha about inline fallback context.

**Script structure:**

1. **Parse args:** Accept `--patch`, `--minor`, `--major` (exactly one required), and `--dry-run` (optional). Print usage and exit 1 on invalid input. No positional args.

2. **Read canonical version:** `jq -r '.metadata.version' "$REPO_ROOT/.claude-plugin/marketplace.json"`. Validate it matches semver pattern `N.N.N` -- fail if not.

3. **Compute new version:** Split with `IFS='.' read -r major minor patch`. Increment per bump type. For minor: reset patch to 0. For major: reset minor and patch to 0.

4. **Collect target files:** Build array of all files to update:
   - `$REPO_ROOT/.claude-plugin/marketplace.json` (1 file, 13 version fields)
   - All `$REPO_ROOT/skills/*/SKILL.md` files (12 files, 1 field each)
   - `$REPO_ROOT/skills/skippy-dev/commands/migrate.md` (1 file, 1 field)

5. **Dry-run mode:** If `--dry-run`, print "Version bump: CURRENT -> NEW (TYPE)" and list all files that would change (relative paths from repo root), then print "No files modified (dry run)." and exit 0.

6. **Execute bump:**
   - marketplace.json: Use `jq --arg v "$NEW" '(.metadata.version = $v) | (.plugins[].version = $v)'` with temp file + mv pattern (NOT redirect to same file).
   - SKILL.md files: Use `sed -i '' "s/  version: $CURRENT/  version: $NEW/"` for each. The two-space indent anchors to YAML frontmatter only.
   - migrate.md: Use `sed -i '' "s/version: $CURRENT/version: $NEW/"` (no indent anchor -- the template instruction format differs).

7. **Verify:** After all updates, count remaining references to old version in production files:
   ```bash
   REMAINING=$(grep -r "$CURRENT" "$REPO_ROOT/.claude-plugin/" "$REPO_ROOT"/skills/*/SKILL.md "$REPO_ROOT/skills/skippy-dev/commands/migrate.md" 2>/dev/null | wc -l | tr -d ' ')
   ```
   Use `skippy_section "Version Bump"`. If REMAINING is 0, `skippy_pass "All 26 locations updated: $CURRENT -> $NEW"`. If >0, `skippy_fail "$REMAINING old version references remain"`. End with `skippy_summary`.

8. **Make executable:** The file must have mode 755 (set via `chmod +x`).

**Do NOT:**
- Create a VERSION file (marketplace.json IS the source)
- Add git tagging (not in requirements -- keep scope minimal)
- Put deploy-service-specific validation in common.sh
- Use GNU sed syntax (`sed -i "..."` without `''`) -- macOS BSD sed requires the empty string arg
