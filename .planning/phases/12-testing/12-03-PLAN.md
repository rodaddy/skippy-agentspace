Create `.github/workflows/test.yml` that runs the full bats test suite on macOS.

Workflow spec:
- name: "Tests"
- triggers: push (all branches), pull_request (main branch)
- Single job: `test` running on `macos-latest`
- Steps:
  1. `actions/checkout@v4` with `submodules: recursive` (pulls bats submodules)
  2. Run bats: `./tests/bats/bin/bats tests/` (runs all .bats files, TAP output)

Do NOT install bats via brew -- the vendored submodule IS the bats binary. The checkout step with `submodules: recursive` handles everything.

Do NOT add brew install steps for bun/jq/git -- macos-latest has brew pre-installed but the tests that need bun will be skipped if bun isn't available (those tests check `command -v bun` themselves). For a cleaner CI, add a brew install bun step.

Final workflow:
```yaml
name: Tests
on:
  push:
    branches: ['*']
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4
        with:
          submodules: recursive

      - name: Install bun
        run: brew install oven-sh/bun/bun

      - name: Run tests
        run: ./tests/bats/bin/bats tests/
```
