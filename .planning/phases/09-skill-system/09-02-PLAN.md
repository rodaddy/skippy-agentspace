Create `skills/skippy/commands/migrate.md` following the update.md AI command pattern. This is the migration tool per user decision (AI command, not shell script).

The command must implement:

**Frontmatter:**
```yaml
---
name: skippy:migrate
description: Migrate PAI skills into portable format
---
```

**Objective:** Scan ~/.config/pai/Skills/, rank by priority criteria (daily driver frequency > foundational chain > portable value), present candidates to user, and migrate approved skills to portable format under skills/.

**Process steps (each is a ## section):**

## 1. Scan Source Skills
- Read all directories under `~/.config/pai/Skills/`
- For each: read SKILL.md (or whatever entry file exists), count lines, identify subdirectory structure
- Classify existing directory types (Workflows/, Tools/, helpers/, templates/, agents/, config/, bin/)
- Note which skills already exist under `skills/` (skip or offer re-migrate)

## 2. Rank and Present Candidates
- Sort by: daily driver frequency (check against CLAUDE.md "Skills You Keep Skipping" table), then foundational chain, then portable value
- Present ranked table to user with columns: Name, Lines, Complexity (LOW/MEDIUM/HIGH), Category suggestion
- User approves, rejects, or adjusts each candidate
- Flag duplicates interactively -- skills that overlap in purpose

## 3. Dry-Run Preview (per skill)
- Show target file tree (skills/<name>/SKILL.md, references/, commands/, scripts/)
- Show SKILL.md preview (first 30 lines) with line count
- Show what gets stripped (PAI-specific references, private content)
- Show what gets flattened (source dirs -> target dirs per Pattern 1 from research)
- Wait for user confirmation before writing

## 4. Migrate (per approved skill)
- Create target directory: `skills/<name>/`
- Create SKILL.md: extract frontmatter (name, description, metadata with category), slim to <150 lines
  - Keep: frontmatter, description, quick reference table, common ops, reference list, top-5 gotchas
  - Move to references/: detailed workflows, extended gotchas, code examples, config details
- Flatten directories per Pattern 1: Workflows/ -> references/, Tools/ -> scripts/ or references/, helpers/ -> scripts/, templates/ -> references/, agents/ -> references/, config/ -> references/
- Sanitize private content: replace IPs matching `10\.71\.\d+\.\d+` with `<your-server-ip>`, domains matching `rodaddy\.live` with `<your-domain>`, credential names with `<your-credential>`, server names with `<your-host>`
- Add "PAI enhancements available" gap notes where PAI-specific features were stripped
- Normalize name to lowercase-kebab (e.g., Art -> art, CreateSkill -> create-skill)
- Skip non-portable files: node_modules, bun.lock, .DS_Store, compiled artifacts

## 5. Update Integration Files
- Add plugin entry to `.claude-plugin/marketplace.json` for each migrated skill
- Run `tools/index-sync.sh --generate` to rebuild INDEX.md
- Report summary: skills migrated, total files created, any warnings

Include execution_context referencing `@../SKILL.md` for the skippy skill context.
