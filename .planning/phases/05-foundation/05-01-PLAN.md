Create CONVENTIONS.md at repo root with four sections:

1. **Content Classification** -- a table listing content types as public or private:
   - Public: Skills (`skills/`), References (`skills/*/references/`), Commands (`skills/*/commands/`), Tools/Scripts (`tools/`, `skills/*/scripts/`), Plugin metadata (`.claude-plugin/`), Upstream tracking (`upstreams/`), Planning artifacts (`.planning/`)

   Include a clear statement: "Private content lives externally at `~/.config/pai-private/`. The repo itself is entirely public-safe. No `private/` directory exists in-repo."

2. **Private Content Location** -- document `~/.config/pai-private/` structure:
   - `memory/` -- MEMORY.md and session state
   - `rules/` -- security protocols, style rules
   - Explain this is the existing PAI pattern, not new

3. **Upstream Registry** -- document the `upstreams/` directory pattern:
   - Each upstream = a directory under `upstreams/` containing `upstream.json`
   - Document the JSON schema fields: name (string, required), description (string, required), repo (URL string, required), branch (string, required), last_checked_sha (string, required -- "none" if never checked), last_check (string, required -- ISO 8601 date or "never"), cherry_picks (string[], optional), notes (string, optional)
   - Explain how to add a new upstream: create directory, copy/fill upstream.json template
   - Note: `ls upstreams/` IS the registry query -- no discovery code needed

4. **Installation Philosophy** -- document the v1.1 shift:
   - Shell scripts ONLY for prerequisite validation (bun, jq, git, bash 4+)
   - All actual install/config/update operations live in markdown instruction files designed for AI agents
   - Existing v1.0 `tools/install.sh` remains for backward compatibility

Keep the document under 120 lines. Use tables for classification. No fluff.
