Create `tools/lib/skippy-state.ts` -- a minimal bun-based parser for reconcile's structured data needs.

The file must:

1. Export a `Task` interface:
```typescript
interface Task {
  number: number;
  name: string;
  files: string[];
  action: string;
  verify: string;
  done: string;
}
```

2. Export `parseFrontmatter(content: string): Record<string, unknown>`:
- Split content on `---` delimiters to extract YAML block
- Parse simple key:value pairs (string, number, boolean)
- Parse arrays (both inline `[a, b]` and multiline `- item` style)
- Parse nested objects (must_haves with sub-keys)
- No external dependencies -- use string splitting and regex
- Return parsed object. Return empty object if no frontmatter found.

3. Export `extractTasks(content: string): Task[]`:
- Match `## Task (\d+): (.+)` pattern for task headings
- For each heading, extract fields from the lines below:
  - `- files:` line -> parse comma-separated paths (strip backticks)
  - `- action:` line -> capture text after prefix
  - `- verify:` line -> capture text after prefix (strip backticks)
  - `- done:` line -> capture text after prefix
- Stop extracting fields when hitting the next `## Task` heading or end of file
- Return array of Task objects

4. Export `classifyTaskStatus(planned: Task[], summaryContent: string): Array<{ task: Task; status: 'DONE' | 'MODIFIED' | 'SKIPPED' | 'ADDED'; evidence: string }>`:
- For each planned task, search summaryContent for evidence of completion
- Look in "## Task Commits" and "## Accomplishments" sections
- Classify: DONE (task name found with matching evidence), MODIFIED (found but different), SKIPPED (not found), ADDED (in summary but not in planned)
- Return classification with evidence string

5. CLI entry point when called directly (`if (import.meta.main)`):
- Accept subcommands: `parse-frontmatter <file>`, `extract-tasks <file>`, `classify-tasks <plan-file> <summary-file>`
- Read files with `Bun.file().text()`
- Output JSON to stdout

Add shebang: `#!/usr/bin/env bun`

Keep the file under 100 lines. No external dependencies (no gray-matter, no yaml library). The YAML frontmatter in .planning/ files uses simple structures that manual parsing handles.
