# Feature Landscape

**Domain:** Portable skill distribution system for Claude Code (personal/small-team marketplace)
**Researched:** 2026-03-06

## Ecosystem Context

The Claude Code skill ecosystem has exploded since Anthropic published the Agent Skills open standard in late 2025. As of March 2026:

- **9,000+ plugins** available in the ecosystem
- **Anthropic's official plugin marketplace** (`anthropics/skills`) ships example and document-processing skills
- **Community marketplaces** (SkillsMP, LobeHub) aggregate thousands of skills
- **Framework-level systems** (GSD, PAUL, OMC) bundle skills into opinionated workflows
- **The Agent Skills open standard** makes SKILL.md portable across Claude Code, Codex CLI, Gemini CLI, Cursor, GitHub Copilot, VS Code, Goose, Roo Code, and 30+ other agents

**skippy-agentspace sits in a specific niche:** it is NOT competing with large marketplaces. It is a personal/PAI-first skill repository that cherry-picks workflow ideas from multiple frameworks (GSD, PAUL, OMC) into standalone, installable skills. The distribution target is one user (Rico) with optional portability to vanilla Claude Code users.

This shapes feature priorities heavily -- infrastructure-grade registry features are anti-features, while personal curation, upstream monitoring, and PAI integration are differentiators.

## Table Stakes

Features users expect. Missing = the tool feels broken or incomplete.

| Feature | Why Expected | Complexity | Notes |
|---------|--------------|------------|-------|
| **SKILL.md with valid frontmatter** | Agent Skills open standard. Every skill system expects `name`, `description` in YAML frontmatter. Without it, skills don't auto-discover. | Low | Current SKILL.md has frontmatter but uses non-standard `triggers` field. Needs alignment: `name`, `description` required; `disable-model-invocation`, `allowed-tools`, `context`, `metadata` optional. |
| **Install / Uninstall** | Users expect a single command to add or remove a skill. Claude Code provides `/plugin install name@marketplace` natively. | Low | Already done via `tools/install.sh` and `tools/uninstall.sh` for manual path. Plugin packaging enables the native `/plugin install` path. |
| **Self-contained skill directories** | Each skill must be a single directory with SKILL.md as entrypoint. No cross-skill imports. Agent Skills spec requires this. | Low | Already done. `skippy-dev/` follows this pattern. |
| **Progressive disclosure** | Claude Code loads ~100 tokens of frontmatter at startup, full content on activation, resources on demand. Skills must be structured for this. | Low | Already done. References are in `references/`, loaded on demand. SKILL.md is ~84 lines. |
| **Works with vanilla Claude Code** | The skill must work when installed to `~/.claude/skills/` or via plugin without any PAI infrastructure. | Low | Already a design constraint. Blocked by hardcoded absolute paths in `commands/reconcile.md`. |
| **Portable paths** | No hardcoded user-specific absolute paths. Skills must work on any machine. | Low | BROKEN -- `commands/reconcile.md` contains `@/Users/rico/.config/pai/...`. Must use `${CLAUDE_SKILL_DIR}` or relative paths. |
| **Skill listing / discovery** | User should see what's available before installing. | Low | Already done -- `install.sh` with no args lists skills. Plugin system provides `/plugin list`. |
| **INDEX.md registry** | Machine-readable index of all skills, commands, references. | Low | Already done. `INDEX.md` auto-generated by `tools/index-sync.sh`. |

## Differentiators

Features that set skippy-agentspace apart. Not expected, but create real value.

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| **Upstream monitoring** (`/skippy:update`) | No other skill system tracks upstream frameworks for new ideas to absorb. This monitors GSD and PAUL repos for changes worth stealing. | Medium | Partially built via `bin/skippy-update.sh`. Needs to clone repos, diff against `.versions`, report meaningful changes. |
| **Cross-framework synthesis** | Every other system is single-framework. skippy-agentspace extracts best ideas from GSD + PAUL + OMC and composes them. | Medium | Already done architecturally. 5 reference docs are PAUL ideas packaged for GSD workflows. |
| **PAI-enhanced, vanilla-compatible** | Dual-mode: works standalone with Claude Code, gains superpowers when PAI infrastructure is present. No other skill system has this layered model. | Medium | Design constraint exists. Works implicitly via CLAUDE.md presence. |
| **Reconciliation** (`/skippy:reconcile`) | PAUL introduced mandatory plan-vs-actual comparison. GSD doesn't have it. No skill marketplace packages it as a portable workflow. | Medium | Command exists. Needs testing against real completed GSD phases. |
| **Plugin marketplace distribution** | Repo doubles as a Claude Code plugin marketplace. Users install via `/plugin marketplace add rodaddy/skippy-agentspace`. | Medium | Not yet built. Need `.claude-plugin/plugin.json` and `marketplace.json`. |
| **Context bracket awareness** | Reference doc teaching agents to self-monitor context window fill and adapt behavior. No other skill addresses context rot. | Low | Already built as `references/context-brackets.md`. |
| **Ephemeral file management** (`/skippy:cleanup`) | Claude Code generates tons of debug logs. No skill marketplace addresses cleanup with quarantine-before-delete. | Low | Already built via `bin/skippy-cleanup.sh`. |
| **Skill authoring scaffold** (`tools/new-skill.sh`) | Generate correct skill directory with SKILL.md template, valid frontmatter, directory stubs. | Low | Not yet built. Defer until second skill is needed. |

## Anti-Features

Features to explicitly NOT build.

| Anti-Feature | Why Avoid | What to Do Instead |
|--------------|-----------|-------------------|
| **npm/bun package publishing** | PROJECT.md says "private repo, manual install." Git-based plugin marketplace achieves distribution without registry. | Git repo as plugin marketplace source. |
| **Web-based marketplace UI** | Multiple community UIs exist. Building one for a personal repo is vanity engineering. | INDEX.md + `/plugin install` is the interface. |
| **Cross-agent portability testing** | Agent Skills format is inherently portable. Testing across 30 agents is unreasonable for solo dev. | Target Claude Code. Format is compatible with others by default. |
| **Auto-merging upstream changes** | PROJECT.md explicitly rejects this. Auto-merge is dangerous for curated content. | `/skippy:update` reports diffs. Human decides. |
| **Skill telemetry / analytics** | Noise for a personal repo. | Use git log. |
| **Dependency resolution between skills** | Self-contained skills only. Inter-skill dependencies add massive complexity. | Duplicate shared knowledge into each skill. |
| **MCP servers inside skills** | MCP is for external connectivity. Skills are for procedural knowledge. Mixing creates dependency hell. | Keep skills as markdown + shell. |
| **TypeScript/Node.js build step** | Adds dependencies, kills "zero-dep, just copy" portability. | Shell scripts + markdown only. |
| **Hooks for behavior enforcement** | Hooks can't detect semantic context. Rules are self-enforced by the LLM reading reference docs. | Reference docs Claude follows, not mechanical triggers. |

## Feature Dependencies

```
SKILL.md frontmatter compliance --> Claude Code auto-discovery
                                --> INDEX.md generation
                                --> Plugin packaging

Portable paths (CLAUDE_SKILL_DIR) --> Plugin packaging (plugins copy files to cache)
                                  --> Cross-machine portability

install.sh --> Skill directory structure
           --> ~/.claude/commands/ symlinks (legacy)
           --> ~/.claude/skills/ symlinks (modern)

Plugin packaging (.claude-plugin/) --> /plugin marketplace add
                                   --> /plugin install

/skippy:update --> .versions tracking
               --> Git access to GSD + PAUL repos

/skippy:reconcile --> GSD .planning/ directory structure
                  --> Phase PLAN.md + execution output

PAI detection --> All skills (optional enhancement)
              --> Independent of vanilla operation
```

## MVP Recommendation

Prioritize:
1. **Path portability** -- Replace hardcoded `/Users/rico/...` with `${CLAUDE_SKILL_DIR}` or relative paths. Without this, nothing else matters for distribution.
2. **Frontmatter alignment** -- Remove non-standard `triggers:` field, add `metadata:` block with version and author.
3. **Rename `bin/` to `scripts/`** -- Agent Skills spec compliance.
4. **Plugin packaging** -- Create `.claude-plugin/plugin.json` and `marketplace.json`.
5. **Reconciliation command validation** -- Test `/skippy:reconcile` against real completed GSD phases.

Defer:
- **Skill scaffolding** (`new-skill.sh`): Build when adding the second skill, not before.
- **Cross-agent testing**: Format is already compatible.
- **Frontmatter validation in CI**: Manual review catches issues at this scale.
- **PAI detection formalization**: Works implicitly via CLAUDE.md presence.
